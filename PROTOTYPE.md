# 王寵服務管理系統 (WANG CHOICE) - 原型開發說明文件 (PROTOTYPE.md)

## 0. 專案概況
本原型旨在驗證「寵物美容/居家消毒」服務的生命週期管理流程。從報價、派工、施工到結案，實現數位化單據流轉與自動化計算。

## 1. 系統架構與目錄結構
專案採用純 HTML/CSS/JS 開發，確保在 `file://` 協議下（無伺服器環境）亦能流暢運作。

```text
wang-choice/
├── index.html          # 專案看板：生命週期監控中心
├── quotation.html      # 報價階段：動態項目選取與金額試算
├── work-dispatch.html  # 派工階段：資源調度、工時規劃與耗材預估
├── work-order.html     # 施工階段：現場進度同步（Preview）
├── completion.html     # 結案階段：掃碼滿意度調查與數位簽章
├── css/
│   └── styles.css      # 全站統一 UI 規範 (含王寵主題色)
├── js/
│   ├── index.js        # 看板狀態渲染邏輯
│   ├── quotation.js    # 報價核心：MASTER_DATA 驅動、金額試算
│   ├── work-dispatch.js# 派工核心：專業施工 Table、動態人員指派
│   └── completion.js   # 結案核心：Canvas 手寫簽章處理
└── img/
    └── qrcode.png      # 滿意度調查 QR Code (連結至 Google 表單)
```

## 2. 核心技術實作與經驗

### 2.1 MASTER_DATA 驅動設計
全站服務項目由統一的 JSON 矩陣驅動（美容室 11 項 / 會客區 5 項）。
- **報價與派工連動**：派工單同步導入報價項目，並優化為「唯讀確認」模式，確保執行合規。

### 2.2 JS 邏輯分化 (Modularization)
完成從單一 `app.js` 到「功能專屬 JS」的演進。每個頁面只載入其需要的邏輯，有效避免了跨頁面 DOM 元素缺失導致的腳本錯誤。

### 2.3 專業施工表格 (Professional Dispatch)
派工單演進為「全欄位可編輯表格」：
- **預計 vs 實際**：結構化區分 [預計時段]、[預計工時] 與 [實際填寫時間]。
- **自定義時段**：支持領班根據現場狀況，直接修改時段名稱或備註。

### 2.4 資源與耗材管理
- **車程規劃**：新增「車程（來回）」參數與計畫施作日對齊。
- **耗材對照**：實作「預估使用量 (預設 2500ML)」與「實際使用量」欄位，為成本分析奠定基礎。

### 2.5 結案與滿意度 (Customer Feedback)
- **掃碼整合**：捨棄傳統紙本或網頁表單填寫，直接在系統中嵌入 **QR Code**。
- **外部串接**：引導客戶掃碼連結至公司統一的 **Google 表單**，確保數據彙整的即時性與準確性。
- **數位簽章**：採用 HTML5 Canvas 實作手寫簽章，支持行動端觸控操作。

## 3. 業務流程亮點
- **進度可視化**：Dashboard 提供燈號與進度點位，呈現案件從報價到結案的完整路徑。
- **資訊傳承**：聯絡資訊、施作地址與重點備註從報價階段自動帶入後續單據。
